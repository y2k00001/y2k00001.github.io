和湛WebRTC平台接口文档 v1.0.0

1 概述

该文档用于描述和湛WebRTC平台为第三方提供webrtc服务的相关接口

2 接口规范

2.1 通信协议

HTTPS协议

2.2 请求方法

所有接口只支持GET或者POST方法

2.3 字符编码

HTTP通讯及报文均采用UTF-8字符集编码格式。

2.4 IP白名单限制

要调用接口的第三方提供IP，由平台添加入进白名单

2.5 请求报文结构

2.5.1 参数说明

Headers(http请求头)：Content-Type:application/json;charset=UTF-8

body(请求体)：其明文为每次请求的具体参数，采用 JSON 格式

2.5.2 请求体明文结构说明

请求body采用JSON格式，其中的参数节点：

参数名称 类型 是否必填 描述
hosp_id string Y 用户登录后token，没有登录则为空字符串
data string N 每个接口的参数json 编码成字符串后再加密形成的字符串
tmestamp int Y 当前UNIX时间戳
sign string Y 请求签名

2.5.3 入参中的data数据加密

  1. 将入参中的data json编码成字符串
  2. 调用openssl_encrypt方法加密data生成的字符串（默认使用AES-128-ECB算法，加密key使用平台分配的token）；
  3. 推荐使用平台提供加密类加密数据

2.5.4 入参中的sign签名生成

  1. 将请求参数数组(hosp_id,加密后的data，timestamp)升序排序
  2. 拼接字符串（空字符串，数字0，null,字符0不参与拼接）
  3. 将平台分配的hosp_id、拼接的参数字符串、平台分配的token拼接后使用sha256算法算出hash值，
  4. 将生成的hash值作为签名放入入参数组中

2.6 响应报文结构

2.6.1 响应报文结构说明

所有接口响应均采用JSON格式，如无特殊说明，每次请求的返回值中，都包含下列字段：

参数名称 类型 是否必填 描述
code int Y 请求响应码，代码定义请见“附录 枚举值 code响应码说明”
message string Y 响应描述
error_code int Y 业务结果码,代码定义请见“附录 枚举值 error_code 业务结果码说明
req_url string Y 请求接口地址
data array N 每个接口特有的参数，详见每个接口定义
req_id int N 请求接口地址
